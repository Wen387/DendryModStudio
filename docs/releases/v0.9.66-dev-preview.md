# v0.9.66 Dev Preview Release Prep

Date: 2026-05-10
Status: local draft preparation for a GitHub pre-release, pending final desktop QA.

## Current Verification

- The root and desktop package versions have been advanced to `0.9.66`.
- The release workflow builds Windows `.exe`, Linux AppImage, and Linux Deb
  assets through the `Desktop Release` workflow.
- Release builds fetch the bundled Python runtime before packaging desktop
  artifacts.
- The bundled update manifest reports `latestVersion: 0.9.66`.
- The normal Windows `dist:win` package now uses the faster `deps-in-asar`
  layout: JavaScript dependencies and parser resources are packed into
  `app.asar`, while Python runtime resources remain loose.

## Release Page and Link Status

Before publishing the preview, run `Desktop Release` manually on the current
`main` with `publish_release` disabled and confirm the uploaded artifacts
include:

- `DendryModStudio-win-x64.exe`
- `DendryModStudio-linux-x64.AppImage`
- `DendryModStudio-linux-x64.deb`

After the artifact-only run passes on the current `main`, rerun the workflow
with `publish_release` enabled and `release_tag` set to `v0.9.66-preview.1`,
or push the same `v*` tag, then confirm the README latest download links
resolve to those Release assets.

## Release Candidate Gate

Run these checks from a fresh clone before attaching packages to a release:

```bash
npm ci --ignore-scripts
npm run check:ci
```

Then run desktop checks:

```bash
cd tools/project_map/desktop
npm ci
npm run smoke
npm run doctor
```

On Linux, a local packaging check can also build the AppImage and Deb:

```bash
npm run dist:linux
```

Windows artifacts should be built by the GitHub `Desktop Release` workflow.
