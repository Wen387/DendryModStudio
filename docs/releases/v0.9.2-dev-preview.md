## Release Candidate Gate

Run these checks from a fresh clone before attaching packages to a release:

```bash
npm ci --ignore-scripts
npm run check:ci
```

Then run desktop checks:

```bash
cd tools/project_map/desktop
npm ci
npm run smoke
npm run doctor
```

On Linux, a local packaging check can also build the AppImage and Deb:

```bash
npm run dist:linux
```

Windows artifacts should be built by the GitHub `Desktop Release` workflow.

Before calling the package ready for invitee testing, manually walk the player
path from a clean app data state:

1. Open the packaged desktop app.
2. Use Quick Start and load the bundled Demo Template.
3. Inspect the demo event, player text, choices, effects, and variable condition.
4. Create one proposal and save it to My Changes.
5. Reopen the saved draft and run Review & Apply dry-run.
6. Apply only guarded operations in a disposable project copy.
7. Open Runtime Preview and confirm the comparison page works.
8. Check the update notice button/banner in both English and Traditional Chinese UI.
